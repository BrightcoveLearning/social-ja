    ソーシャル解析を確認する

    このトピックでは、Brightcove Social を使用して動画のソーシャルパフォーマンスを監視する方法について説明します。

    Brightcove Social では、ソーシャルメディアアカウントに投稿された動画のパフォーマンスを監視するために使用できるレポートがいくつかあります。

    • ダッシュボード -ソーシャルメディアアカウントの動画パフォーマンスの概要を概説します。このトピックで説明しています。
    • ソーシャルビュー -ソーシャルメディアアカウントで共有された動画のソーシャルビューの概要を提供します。詳しくは、ソーシャルビュー分析のレビューを参照してください。
    • ソーシャルエンゲージメント -ソーシャルメディアアカウントで共有された動画のソーシャルエンゲージメント（いいね、共有、リツイートなど）の概要を提供します。詳細については、「ソーシャルエンゲージメント分析のレビュー」を参照してください。

    まず、期間を選択します。レポートを選択するには、ページの上部にあるタブをクリックします。ダッシュボードはデフォルトで表示されます。

    ダッシュボードは、次のセクションで構成されています。

    • ビデオビューを一目で確認
    • トップの目的地
    • トップビデオ
    • 動画視聴の傾向

    期間を選択する

    レポートの期間を指定するには、クイック日付をクリックするか、カレンダーコントロールを使用します。ソーシャル分析データは、常に GMT タイムゾーンを使用します。

    クイック日付を使用する

    一連のクイック日付がページの上部に表示されます。次のクイック日付を使用できます。

    • 過去7日間
    • 過去30日間
    • 先月
    • 2ヶ月前

    レポートを更新するには、クイック日付をクリックします。

    カレンダーコントロールの使用

    カレンダーコントロールをクリックして、開始日と終了日を選択することもできます。

    [ 適用 ] をクリックして、レポートを更新します。

    ビデオビューを一目で確認

    [動画再生数] セクションには、選択した期間の Video Cloud プレーヤー、Facebook、YouTube からの動画再生回数、および合計再生回数が表示されます。更新日は、Video Cloud がソーシャル分析データを保持している最新の日付です。通常、ソーシャルプラットフォームが API のビューとエンゲージメントデータを更新する前に、24 ～ 48 時間以上の遅延があります。

    注：Facebook アナリティクスの [動画で投稿を作成] ボタンをクリックすると、Brightcove Social が作成した動画に新しい二次投稿が作成され、2 つの同一の動画投稿が表示されます。ブライトコーブ Social Analytics では、元のデータのみが表示されます。
    注：Twitter Insights API はメディアビューをレポートしないため、データの閲覧はソーシャルアナリティクスの一部として使用できません。

    [ マイサイトとアプリ ] リンクをクリックすると、Video Cloud Analytics パフォーマンスレポートが開きます。

    視聴回数は、ソーシャルメディアプラットフォームごとに異なってカウントされます。

    • 個人用サイトとアプリ -Video Cloud は、動画の再生を開始したときに動画ビューをカウントします
    • Facebook -視聴者が動画を 3 秒以上視聴したときにカウントされる動画視聴回数
    • YouTube -視聴者が動画を「約」30 秒間視聴したときにカウントされる動画視聴回数

    トップの目的地

    [Top Destations] セクションには、選択した期間の視聴回数が多い上位 5 つの目的地が表示されます。

    動画のタイトルをクリックすると、その動画のソーシャルビュー分析が開きます。

    トップビデオ

    [上位の動画] セクションには、選択した期間の視聴回数が多い上位 5 つの動画が表示されます。

    動画のタイトルをクリックすると、その動画のソーシャルビュー分析が開きます。

    動画視聴の傾向

    動画視聴数のトレンドグラフには、選択した期間におけるソーシャルメディアアカウントに投稿された動画の動画再生数が表示されます。グラフ内の線の上にマウスポインタを置くと、その日付のデータが表示されます。

    注：Twitter Insights API はメディアビューをレポートしないため、データの閲覧はソーシャルアナリティクスの一部として使用できません。

    グラフの右側の凡例を使用して、表示する詳細レベルを選択できます。

    • マイサイトとアプリ -Video Cloud プレーヤーとアプリからの動画再生数を表示します
    • Facebook-Facebookの動画再生数を表示する
    • YouTube-YouTubeの動画再生数を表示する
    • Total -すべてのプラットフォームでのビデオ再生の総数を表示します

    アナリティクスレポートに表示される YouTube の視聴回は、 YouTube アナリティクス概要レポートから収集されます。YouTube は、このデータがリアルタイムデータであることを保証するものではありません。そのため、このレポートに視聴回数やエンゲージメントが表示されるまでに数日遅れることがあります。データは YouTube アナリティクスに保存されると、ブライトコーブソーシャルアナリティクスの夜間のデータコレクターによって収集されます。

    また、最新のリアルタイム視聴データが必要な場合は、最近公開した上位 25 本の動画について、過去 48 時間の視聴データを見積もったリアルタイムレポートも用意されています。YouTube チャンネルのリアルタイムレポートはこちらからご覧いただけます。Brightcove Social Analytics はこのリアルタイムデータを収集しません。これは、実際の動画ではなく、推定視聴回数を提供し、最近の動画にのみ適用されます。

    削除された動画

    削除された動画のソーシャル分析の表示方法は、動画が配信された方法と、動画が AutoSync の場合は AutoSync 設定の内容によって異なります。

    • 動画がソーシャル公開機能を使用して配信された場合、Video Cloud から削除しても動画はソーシャルプラットフォームから削除されず、Social は引き続き動画の分析を収集します。
    • AutoSync 機能を使用してビデオが配信された場合、自動同期の設定によっては、Video Cloud から削除すると、ソーシャルプラットフォームから動画が削除される場合があります。
      • [< ソーシャルプラットフォームで配信動画を削除する >] オプションが選択されている場合、ソーシャルモジュールはソーシャルプラットフォームから動画を削除し、分析データの収集を停止します。
      • ソーシャルプラットフォームに動画を残すオプション（これらのオプションの正確な詳細はプラットフォームによって異なります）が選択された場合、Social モジュールは引き続きその分析の収集を試みます。
    • Social Analytics では、削除された動画の既存のソーシャル分析データが引き続き表示されます。
